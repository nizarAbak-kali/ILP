Ensemble de paquetages Java associés à quelques autres programmes en d'autres langages (Makefile, RelaxNG, Scheme) liés à l'unité d'enseignement « Implantation d'un Langage de Programmation » dispensé à l'UPMC (Université Pierre et Marie Curie).

Ces paquetages décrivent des interprètes et des compilateurs pour une série de langages emboîtés de la classe de JavaScript mais dont la syntaxe concrète est XML. Le compilateur est écrit en Java et produit du C. Le but de tous ces différents langages est de souligner les différents langages impliqués (syntaxe d'entrée, langage d'entrée, langage de sortie et langage de la bibliothèque d'exécution, langage dans lequel est écrit le compilateur) ce qui ici est plus simple qu'un compilateur de Scheme en Scheme comme dans « Principes d'implantation de Scheme et Lisp » [Queinnec2007].

Le paquetage ilp1.interfaces contient les interfaces décrivant des AST (arbres de syntaxe abstraite). Ces interfaces sont associées à des grammaires appropriés des schémas RelaxNG permettant de valider les programmes représentés par des fichiers XML.

Le paquetage ilp1.fromxml contient un premier analyseur syntaxique XML vers AST.

Le paquetage ilp1.eval contient un analyseur syntaxique XML vers EAST amélioré ainsi qu'un interprète.

Le paquetage ilp1.cgen contient un analyseur syntaxique XML vers CEAST et un compilateur vers C. La bibliothèque d'exécution en C se trouve dans le répertoire C.

Le paquetage ilp2 étend le langage pour inclure l'affectation, la boucle tant que et les définitions de fonctions globales. Il reprend le meilleur des paquetages précédents (afin de ne plus en dépendre) et définit un interprète et un compilateur vers C. Ce paquetage contient également une nouvelle bibliothèque d'exécution pour l'interprète en Java.

Le paquetage ilp3 étend le langage ILP2 pour inclure le traitement des exceptions. Il ne fait que complémenter le paquetage précédent.

Le paquetage ilp4 modifie le langage ILP3 afin que toute instruction soit également une expression. Cette modification permet d'implanter une optimisation: l'intégration (ou inlining). Le paquetage ne dépend pas du précédent.

Le paquetage ilp6 étend le langage ILP4 avec des classes et des objets. Ce paquetage ne dépend pas du précédent.

Ces programmes ont été développés sur Linux avec Emacs, make, jikes, RelaxNG ainsi qu'Eclipse.
